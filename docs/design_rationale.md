# AIRPG 技術選型與設計決策 (Design Rationale)

這份文件詳述了本專案在開發過程中的技術選型考量，旨在說明各項決策背後的工程思維。

---

## 1. 後端框架：FastAPI
**決策原因：** 
*   **高效能 (High Performance)**：FastAPI 基於 Starlette 和 Pydantic，是當前 Python 最快的 Web 框架之一，其非同步 (async/await) 支援非常適合處理 AI 推論等 I/O 密集型任務。
*   **類型安全 (Type Safety)**：深度整合 Python Type Hints，能大幅提升開發效率並減少運行時錯誤。
*   **自動化規格文件**：原生支援 OpenAPI (Swagger)，讓前端 (Godot) 在對接 API 時有明確的標準參考，減少溝通成本。
*   **現代化設計**：相比 Flask 的精簡或 Django 的沉重，FastAPI 提供了最佳的平衡點，適合快速迭代開發。

## 2. ASGI 伺服器：Uvicorn
**決策原因：**
*   **非同步協定支援**：作為解決 Python 非同步處理的標準實作，Uvicorn 能發揮 FastAPI 所有的效能潛力。
*   **輕量與穩定**：它是開發與部署 FastAPI 應用的標準選型，在處理併發請求時表現優異。

## 3. 核心技術：RAG (Retrieval-Augmented Generation)
**決策原因：**
*   **消除幻覺 (Hallucination Mitigation)**：通用 LLM 往往無法得知特定遊戲的世界觀細節。透過 RAG，我們將真實的「世界觀集」注入 Prompt，確保 AI 角色的發言有據可查。
*   **知識更新彈性**：無需重新微調 (Fine-tuning) 模型，只需更換 `server/data/` 下的主題文件，即可讓 AI 瞬間切換不同的知識背景，大幅降低維護成本。
*   **隱私與數據控制**：所有參考資料都存放在本地向量庫，確保敏感的世界觀設定不會流向第三方平台。

## 4. 模型推理：Ollama (本地 LLM)
**決策原因：**
*   **隱私與安全性**：本專案選用 Ollama 作為推論引擎，核心考量是確保對話數據不離開使用者機器，完全保障個人隱私。
*   **零成本開發**：避免了頻繁呼叫商業 API (如 OpenAI) 產生的費用，且在無網路環境下依然能正常運作。
*   **開發者體驗**：Ollama 將複雜的模型部署簡化為 `pull` 指令，並提供標準的 API 接口，讓在地端模擬雲端 AI 服務變得極其簡單。

## 5. 向量資料庫：ChromaDB
**決策原因：**
*   **輕量級與嵌入式**：ChromaDB 無需安裝繁瑣的獨立資料庫服務，可直接作為 Python 庫運行並持久化至本地目錄，完美符合「一鍵啟動」的開發目標。
*   **與 LangChain 高度整合**：其 API 簡潔，在處理知識庫檢索時表現穩定。

## 6. 編排框架：LangChain (LCEL)
**決策原因：**
*   **模組化設計**：LangChain 提供了標準的組件化接口，讓我們能輕易抽換不同的 LLM 或 VectorStore。
*   **LCEL (LangChain Expression Language)**：我們採用了 LCEL 語法來串接 RAG 流程，這讓邏輯從「黑盒」變得清晰、可讀且易於調試。

## 7. 前端引擎：Godot Engine 4.4
**決策原因：**
*   **沉浸式 UI 潛力**：相比傳統 React/Vue 網頁，Godot 作為遊戲引擎能提供極其豐富的骨架動畫、特效與粒子系統，這對於「角色扮演」系統至關重要。
*   **Web 跨平台實力**：Godot 優秀的 Web Export 能力讓它具備網頁的便利性，同時擁有遊戲的性能。

---

## 8. 整合式 Launcher Dashboard
**決策原因：**
*   **降低進入門檻**：將複雜的 CLI 啟動與環境變數編輯轉化為直觀的 Web 介面。
*   **後端進程生命週期管理**：Launcher 負責伺服器的啟停與日誌擷取，確保穩定性。
*   **多雲端供應商抽象化**：系統設計了動態載入機制，使用者可無縫切換 Ollama、OpenAI、Anthropic 與 Google，使專案具備高度的靈活性與商業展示價值。

## 總結：架構核心哲學
本專案的設計哲學是「**專業、自動、透明化**」。
我們不僅追求 AI 技術的應用，更重視**使用者開發體驗 (DX)** 與**系統的可擴展性**。這種「控制台 + API Server + 前端」的架構，是現代化 AI 軟體產品的標準範式。
