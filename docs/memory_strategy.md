# 記憶系統導入時機建議

在 AI RPG 系統中，「記憶」通常分為兩個層次。為了確保開發流程順暢且能及早驗證功能，建議採取**二階段導入**策略：

## 1. 第一階段：對話歷史 (短期記憶/Session Memory)
*   **建議時機**：**現在 (第二階段) - 實作 `/chat` 端點時同步導入**。
*   **與 RAG 的差異**：RAG 提供的是「世界設定」，對話歷史提供的是「玩家剛剛說過什麼」。
*   **原因**：RPG 講求連貫性。如果 AI 記不住上一秒的問題，玩家感官會大打折扣。
*   **實作規劃**：
    *   在 FastAPI 中使用 Pydantic 模型定義 `ChatHistory`。
    *   串接 LangChain 的 `ChatMessageHistory` 管理。

## 2. 第二階段：長期記憶 (Persistent / Long-term Memory)
*   **建議時機**：**端到端流程穩定後 (第三階段)**。
*   **特色**：跨 Session、跨越天數的記憶。涉及將重要玩家行為存入 SQL 或另一個專屬的「行為向量庫」。
*   **原因**：需要結合 LLM 摘要（Summarization）邏輯，且數據庫存儲結構較複雜。

---

## 總結建議
不要等到最後才開始做記憶。**現在就應該在實作 `/chat` API 的同時，加入基礎的對話歷史紀錄功能**，這對提升 AI 的「角色感」最為顯著。

> [!TIP]
> **開發優先順序：**
> 1. 先讓 AI 能根據世界觀回答 (RAG)。
> 2. 再讓 AI 記住當前對話內容 (短期)。
> 3. 最後讓 AI 記住玩家的過往事蹟 (長期)。
